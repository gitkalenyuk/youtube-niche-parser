🔍 Як це працює

Програма НЕ використовує твій акаунт і НЕ ходить через твій особистий браузер. Браузер (Chrome/Edge) тут лише малює це вікно — не більше.

Самі запити до YouTube робить Python — звичайними анонімними запитами, як відвідувач у режимі інкогніто. Без логіну, без кукі, без твоєї історії.

Дані беруться з публічної сторінки пошуку YouTube (вшитий у неї JSON) + публічного внутрішнього API для «гортання» далі за першу сторінку.

⏳ Чи злетить з часом?

Не через акаунт — його нема, тож нічого не «протухає» (на відміну від проксі до AI).

Єдине, що може зламати — якщо YouTube змінить структуру своєї сторінки. Тоді треба буде підправити парсер. Поки формат стабільний — працює.

Тимчасовий ризик: забагато запитів з одного IP → YouTube покаже капчу/тимчасовий блок. Це минає (не бан акаунта). Лікується паузою, меншою к-стю потоків або зміною IP/VPN.

🎯 Фільтри ніші

  • Макс. підписників — показати лише малі канали (недооцінені ніші).
  • Мін. переглядів — відкинути «мертві» відео.
  • V/S (💎) — перегляди ÷ підписники. ≥2× на малому каналі = самородок (відео завірусилось відносно розміру каналу — головний сигнал ніші).
  • Тип/Тривалість/Дата/Характеристики — те саме, що фільтри пошуку на самому YouTube.

⚡ Потоки (швидкість ↔ ризик)

1–3 — безпечно. 4–5 — помірний ризик. 6–8 — швидко, але вищий шанс тимчасового блоку. Для 100+ запитів постав 2–3 і йди пити каву.

💡 Поради по налаштуванню

  • Дата «цього тижня» + сортування «перегляди» → свіжі вірусні теми.
  • Менший «Ліміт на запит» (20–30) = швидше і менше навантаження.
  • Сортуй таблицю по колонці V/S → одразу видно 💎 самородки.
  • Експорт CSV → відкрий в Excel/Sheets для глибшого аналізу.
  • Для Shorts підписники не показуються (їх нема у видачі Shorts).
